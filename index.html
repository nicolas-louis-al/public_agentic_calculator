Understanding the Estimation Methodology

The Agentic Workflow Cost Estimator calculates the financial impact of deploying autonomous coding agents. Unlike simple chat interfaces, agentic workflows involve continuous loops of reading code, writing tests, debugging, and generating features. The footprint of these loops varies dramatically based on your project context and the specific tasks selected.

1. Project Context: Greenfield vs. Brownfield

The total size of your codebase dictates how many tokens are required for comprehensive tasks (like Architecture Scaffolding or Quality Audits).

Greenfield (New Projects)

Starts with 0 Base LOC. The projected repository size is calculated purely dynamically by summing up the sizes of the Feature Implementation tasks you select in the WBS. Repo-wide tasks are evaluated against this newly generated footprint.

Brownfield (Existing Projects)

Starts with a Legacy LOC provided by you. As you add features, we calculate the newly generated LOC. You specify an interactive Code Impact Distribution (New / Update / Delete) using the slider. New code grows the repo size. Updates leave it unchanged. Deletions actively reduce the total repo size.

Proj. LOC = Legacy LOC + (FeatureLOC * %New) - (FeatureLOC * %Delete)

2. Scoped Evaluation Mechanics

Tasks are evaluated differently depending on their semantic scope. Agents do not read a massive 10M LOC repository to write a small 200 LOC feature, but they do read the entire repository when executing a massive Migration task.

A Feature Tasks (Scoped Context)

Feature Implementation tasks are evaluated strictly on their individual LOC size and quantity, ignoring the total repo size. Standard scope sizes:

  • Small Feature: 250 LOC scope
  • Medium Feature: 1,000 LOC scope
  • Large Feature: 5,000 LOC scope
  • XL Epic: 15,000 LOC scope
  • PR Review: 500 LOC diff scope per PR
Base Tokens = FeatureScopeLOC * Density

B Repo Tasks (Global Context)

Tasks like Quality Audits, Legacy Migrations, and Sprint Planning require sweeping the entire codebase. These tasks derive their baseline footprint from the Projected Total Repo Size calculated in Step 1.

Base Tokens = TotalRepoLOC * Density

3. Friction & Discovery Tokens

Agentic coding isn't linear. An agent doesn't read the entire context on every turn. The estimator applies a Friction Factor (based on codebase complexity) to adjust the total turns. It also assumes 25% of the total agent turns are spent "discovering" (reading the full context), while the remaining 75% are "working" turns reading only a fraction (15%) of the context.

Adj. Multiplier (Turns) = Base Multiplier × Friction Factor
Discovery Tokens = Base Tokens × (Adj. Multiplier × 0.25)
Working Tokens = (Base Tokens × 0.15) × (Adj. Multiplier × 0.75)
Gross Tokens = Discovery Tokens + Working Tokens

4. Token Flow & Caching Distribution

Once the Gross Tokens are calculated for a task, they are distributed into three distinct buckets based on the task's output profile and the prompt cache hit rate. Agentic loops rely heavily on reading the same context repeatedly, leading to high cache rates.

Output Tokens = Gross Tokens × Task Output Profile %
Total Input Tokens = Gross Tokens - Output Tokens
Cached Input Tokens = Total Input Tokens × Cache Read Rate %
New Input Tokens = Total Input Tokens - Cached Input Tokens

5. Final Financial Cost Calculation

Finally, the application applies the specific pricing tier of the selected LLM. Because model pricing is quoted per 1 Million (1M) tokens, the tokens are divided accordingly and multiplied by their respective input, cached, and output rates.

CostNew Input = (New Input Tokens / 1,000,000) × Model Input Price
CostCached = (Cached Input Tokens / 1,000,000) × Model Cached Price
CostOutput = (Output Tokens / 1,000,000) × Model Output Price
Base Cost = CostNew Input + CostCached + CostOutput

Stochastic Range Estimation

Because LLM behavior is inherently unpredictable, the final display provides a probabilistic range based on the Base Cost. We estimate a low-end (happy path, fewer turns, high caching) at -20%, and a high-end (hallucination loops, cache eviction) at +45%.
